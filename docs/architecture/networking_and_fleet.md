# Сетевая архитектура и интеграция (Networking & Fleet Management)

Данный документ описывает конфигурацию сетевого стека робота MONA внутри контейнеризированной среды, выбор реализации DDS и стратегию интеграции с системами управления флотом (Fleet Management Systems).

## 1. Изоляция и Сетевой стек Docker
Для минимизации задержек (latency) и обеспечения прозрачного обнаружения узлов (DDS Discovery) контейнер разработки и исполнения настроен на использование сетевого стека хоста:
* **`network_mode: host`**: Контейнер не использует изолированный мост Docker. Он напрямую делит сетевые интерфейсы (eth0, wlan0) с хост-машиной. Это критически важно для корректной работы мультикаста FastRTPS.
* **`ipc: host` / `pid: host`**: Необходимы для использования разделяемой памяти (Shared Memory) между узлами, что радикально ускоряет передачу больших массивов данных (PointClouds, Image).

## 2. Управление областью видимости (DDS)
В проекте используются переменные окружения для гибкого управления областью видимости робота без пересборки контейнера:
* **`RMW_IMPLEMENTATION=rmw_fastrtps_cpp`**: Жесткая фиксация реализации DDS (eProsima Fast DDS). Обеспечивает предсказуемость, высокую производительность и совместимость со стандартами индустрии.
* **`ROS_LOCALHOST_ONLY`**:
    * `1` (Режим разработки): Трафик DDS замыкается внутри петлевого интерфейса (loopback). Защищает локальную Wi-Fi сеть от перегрузки тяжелыми топиками (TF, PointCloud) при локальной симуляции.
    * `0` (Режим интеграции): Мультикаст разрешен. Робот может обмениваться данными с внешним сервером Rviz, пультами управления или другими роботами в локальной сети.

## 3. Интеграция с Fleet Management (Roadmap)
В соответствии с промышленными стандартами логистики, MONA проектируется с учетом будущей поддержки протоколов верхнего уровня для управления флотом:
### VDA 5050
Стандарт ассоциации автомобильной промышленности Германии для обмена данными между AGV/AMR и мастер-контроллером.
* **Архитектурное решение:** Будет реализован MQTT-мост (VDA 5050 Adapter), который транслирует JSON-структуры VDA (Order, InstantActions, State) во внутренние топики ROS 2 (`Nav2 Waypoint Follower`, `cmd_nav`, `robot_status`).
### Open-RMF (Robotics Middleware Framework)
Система управления гетерогенными флотами от Open Robotics.
* **Архитектурное решение:** На стороне робота будет запущен `rmf_fleet_adapter`, который берёт на себя переговоры о маршрутах и избегание конфликтов (Traffic Management) на перекрёстках склада.
