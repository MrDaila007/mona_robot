# Архитектура Lifecycle (Managed Nodes)
В проекте MONA критически важные узлы (драйверы моторов, Safety) реализуются как **Managed Nodes** (управляемые узлы). В отличие от обычных нод, они имеют конечный автомат состояний (State Machine).
## Философия состояний
Это позволяет гарантировать предсказуемое поведение при запуске и аварийной остановке.
1.  **Unconfigured**: Нода создана, но ресурсы не выделены.
    * *Действие:* Конструктор только объявляет параметры.
2.  **Inactive**: Нода сконфигурирована, память выделена, паблишеры созданы, но **данные не публикуются**.
    * *Метод `on_configure`*: Загрузка весов нейросетей, открытие портов, создание `create_publisher`.
3.  **Active**: Рабочий режим. Данные идут в сеть.
    * *Метод `on_activate`*: Включение паблишеров, старт таймеров.
4.  **Finalized**: Завершение работы.
    * *Метод `on_cleanup`*: Очистка памяти, `reset()` умных указателей.
## Launch и автоматизация
Для автоматического запуска используется механизм событий (`EventHandlers`) в Python Launch-файлах.
### Сценарий запуска:
1.  Launch-файл запускает процесс (`OnProcessStart`).
2.  Генерируется событие `TRANSITION_CONFIGURE`.
3.  При успешном переходе в `Inactive` генерируется событие `TRANSITION_ACTIVATE`.

Подробнее см. пример реализации в пакете `mona_safety`.
