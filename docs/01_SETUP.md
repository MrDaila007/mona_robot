# Настройка окружения разработки (Environment Setup)

## Цель
Создать воспроизводимую среду разработки (Dev Environment), базирующуюся на ROS 2 Humble. Среда поддерживает кросс-компиляцию, прямой доступ к "железу" и горячую подмену кода.
## Требования
* Docker Engine (24.0+)
* Docker Compose plugin
* Git
## Архитектура образа
Используется официальный образ **`ros:humble-ros-base`**.
Это "золотая середина" для робота: включает `core` + базовые инструменты (геометрия, TF2, URDF), но без тяжелых GUI-утилит, что позволяет оптимизировать размер.
### Структура проекта
```text
MONA_ws/
├── src/                # Исходный код пакетов (хранится на хосте, монтируется в Docker)
├── Dockerfile          # Описание сборки образа
└── docker-compose.yml  # Оркестрация запуска и параметры сети
```
### Управление контейнером
Запуск среды:
```bash
docker compose up -d --build
```
- Флаг `-d` запускает контейнер в фоне.
- Флаг `--build` пересобирает образ, если Dockerfile изменился.
### Вход в контейнер:
```bash
docker exec -it mona_dev bash
```
### Остановка:
```bash
docker compose down
```
### Особенности конфигурации
В `docker-compose.yml` настроены следующие параметры для доступа к железу:
- `network_mode`: host: Убирает изоляцию сети (необходимо для ROS 2 Discovery / DDS).
- `privileged: true`: Дает доступ ко всем устройствам в /dev (Lidar, STM32, Camera).
- `ipc: host`: Позволяет использовать Shared Memory для быстрой передачи сообщений (Zero Copy).
