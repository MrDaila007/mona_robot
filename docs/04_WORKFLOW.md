# Ежедневный Рабочий процесс (Workflow)

> **Build, Test, Repeat**
> 
> Стабильность системы обеспечивается регулярным тестированием и строгой гигиеной сборки.

## 1. Сборка проекта (Build)
Используется утилита `colcon`.
### Инкрементальная сборка (для быстрой отладки)
```bash
colcon build --symlink-install
```
### Чистая сборка (Clean Build)
Обязательна перед слиянием веток или при странном поведении системы. Используйте предоставленный скрипт:
```bash
./scripts/run_sim.bash
```

## 2. Обновление окружения
После каждой сборки необходимо обновлять переменные окружения:
```bash
source install/setup.bash
```

## 3. Локальная интеграция (Local CI)
Перед созданием коммита запустите скрипт проверки качества:
```bash
./scripts/local_ci.bash
```
**Скрипт выполняет:**
1. Автоформатирование кода (`uncrustify`, `black`).
2. Линтинг (`cpplint`, `flake8`).
3. Запуск тестов.

## 4. Тестирование
Запуск модульных и интеграционных тестов:
```bash
colcon test
colcon test-result --verbose
```

## 5. Управление зависимостями (rosdep)
### Зачем это нужно в C++?
В отличие от Python, где зависимости изолируются в виртуальном окружении, C++ зависимости в ROS 2 - это **системные библиотеки** (заголовочные файлы `.hpp` и бинарные `.so` в `/usr/lib`).
Когда вы добавляете `<depend>lifecycle_msgs</depend>` в `package.xml`, CMake не скачивает библиотеку сам. Он ожидает, что она уже установлена в системе. Утилита `rosdep`:
- Читает `package.xml`.
- Видит там `<depend>lifecycle_msgs</depend>`.
- Идет в свою базу данных и смотрит: "В Ubuntu этот пакет называется `ros-humble-lifecycle-msgs`".
- Вызывает `sudo apt-get install ros-humble-lifecycle-msgs`.
### Когда запускать?
Эту команду нужно выполнять, если:
1. Вы клонировали проект в первый раз.
2. Вы (или ваш коллега) добавили новую библиотеку в `package.xml` и проект перестал собираться с ошибкой вида `Could not find a package configuration file provided by...`.
### Команда
```bash
# --from-paths src: сканировать папку src/
# --ignore-src: не пытаться установить пакеты, которые мы сами же и разрабатываем
# -r: продолжать установку даже при ошибках
# -y: отвечать "yes" на все вопросы apt-get

rosdep update && rosdep install --from-paths src --ignore-src -r -y
```

## 6. Типичные проблемы (Troubleshooting)
### WARNING: colcon.colcon_ros.prefix_path.ament
* **Симптом:** Множество желтых предупреждений при сборке.
* **Причина:** Вы удалили папки `build/install` в активном терминале.
* **Решение:** Просто игнорируйте или перезапустите терминал.
### CMake Error: ament_cmake_symlink_install_directory
* **Симптом:** Ошибка копирования папок (launch, config).
* **Причина:** В `CMakeLists.txt` указана папка, которой нет в проекте.
* **Решение:** Удалите лишнюю строку `install(DIRECTORY ...)` в CMakeLists.txt.
